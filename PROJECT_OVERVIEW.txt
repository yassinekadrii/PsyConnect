# PsyConnect - Technical Project Overview

## 📂 Project Summary
PsyConnect is a comprehensive tele-psychology platform designed to connect patients with qualified specialists. It features real-time messaging, video consultations, medical prescriptions, and a robust administrative dashboard.

---

## 🛠️ Technology Stack
*   **Backend**: Node.js & Express.js
*   **Database**: MongoDB (Mongoose ODM)
*   **Frontend**: HTML5, Vanilla CSS3, Vanilla JavaScript (No heavy frameworks)
*   **Real-time Communication**: Socket.io & WebRTC

---

## 📚 Libraries & Dependencies

### Backend (npm packages)
*   `express`: Core web application framework.
*   `mongoose`: Object Data Modeling (ODM) for MongoDB.
*   `jsonwebtoken (JWT)`: Secure role-based authentication.
*   `bcryptjs`: Industry-standard password hashing.
*   `socket.io`: Real-time bidirectional event-based communication.
*   `cors`: Handling Cross-Origin Resource Sharing.
*   `dotenv`: Environment variable management.
*   `nodemon` (Dev): Automatic server restart during development.

### Frontend (CDN/Internal)
*   `html2pdf.js`: Generates professional PDF prescriptions and statistics reports.
*   `socket.io-client`: Enables real-time chat and video call signaling.
*   `Google Fonts (Inter)`: Modern typography for a premium look.

---

## ⚙️ Core Architecture & Functions

### 🌍 Internationalization (i18n)
*   **File**: `public/js/i18n.js`
*   **Functions**: 
    *   `initI18n()`: Loads translations and applies stored preference.
    *   `applyLanguage(lang)`: Updates UI text and handles RTL (Right-to-Left) layout for Arabic.
    *   `toggleLanguage()`: Cycles through EN, FR, AR with a single button.
*   **Dictionary**: `public/translations.json` (EN, FR, AR).

### 📹 Real-time Console & Video
*   **File**: `public/js/webrtc.js`
*   **Functions**:
    *   `startCall()`: Initializes WebRTC peer connections.
    *   `handleSignaling()`: Uses Socket.io to swap SDP offers/answers and ICE candidates.
*   **Features**: Dynamic video UI, mute/unmute, and call ending.

### 🔐 Authentication & Roles
*   **Controller**: `controllers/authController.js`
*   **Roles**:
    *   **Patient**: Self-registration, browsing doctors, chatting, viewing prescriptions.
    *   **Doctor**: Access to patient list, real-time consultation, issuing prescriptions.
    *   **Admin**: Platform statistics, managing doctors/patients/admins, PDF reporting.

### 📝 Prescription & Reporting
*   **Controller**: `controllers/prescriptionController.js`
*   **Functions**:
    *   `createPrescription()`: Saves medical data and links to patient/doctor.
    *   `downloadPrescriptionPDF()`: Generates professional PDF with medicine notes.

### 📊 Admin Dashboard
*   **Controller**: `controllers/adminController.js`
*   **Features**:
    *   Real-time statistics (totals for users, messages, prescriptions).
    *   Doctor creation & management.
    *   Complete Patient directory.
    *   Platform-wide activity reporting.

---

## 💾 Database Models (Mongoose)
1.  **User**: Stores identity, role, hashed passwords, and profile data.
2.  **Message**: Stores chat history between doctors and patients.
3.  **Prescription**: Stores medical data, medicine arrays, and general instructions.

---

## 📄 Key Pages
*   `index.html`: Public landing page & Specialist directory.
*   `login.html` / `register.html`: Secure entry points.
*   `admin-dashboard.html`: Command center for platform owners.
*   `doctor-chat.html` / `patient-chat.html`: Interactive consultation workspaces.
*   `profile.html`: Secure personal settings and password management.
