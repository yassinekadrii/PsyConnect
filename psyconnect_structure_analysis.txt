# PsyConnect - Project Structure Analysis

## 1. Libraries & Dependencies (package.json)
The project is a Node.js backend using the following key dependencies:
- **Core Framework**: `express` (Web framework), `mongoose` (MongoDB object modeling)
- **Real-time & Communication**: `socket.io` (WebSockets), `nodemailer` (Email sending)
- **Security**: `bcryptjs` (Password hashing), `jsonwebtoken` (Auth tokens), `helmet` (HTTP headers), `express-rate-limit` (DDoS protection), `mongo-sanitize` (NoSQL injection prevention)
- **Utilities**: `dotenv` (Environment variables), `cors` (Cross-origin), `multer` (File uploads)

## 2. Platform Pages (Frontend in `public/`)
These are the HTML files served to users:
- `index.html` (Landing page)
- `login.html` (User authentication)
- `register.html` (User registration)
- `profile.html` (User profile management)
- `admin-dashboard.html` (Admin interface)
- `patient-dashboard.html` (Patient main area)
- `patient-chat.html` (Patient messaging interface)
- `patient-appointments.html` (Patient booking/schedule)
- `doctor-chat.html` (Doctor messaging interface)
- `doctor-schedule.html` (Doctor appointments/availability)

## 3. API Routes (`routes/` & `server.js`)
Base URL path: `/api/`
- **/auth** (`routes/auth.js`): Authentication endpoints (login, register, etc.)
- **/admin** (`routes/admin.js`): Administrative endpoints (user management, stats)
- **/patient** (`routes/patient.js`): Patient-specific actions
- **/doctor** (`routes/doctor.js`): Doctor-specific actions
- **/user** (`routes/user.js`): General user settings and profile
- **/messages** (`routes/messages.js`): Chat and messaging history
- **/prescriptions** (`routes/prescriptions.js`): Medical prescriptions handling
- **/posts** (`routes/posts.js`): Forum or blog posts
- **/appointments** (`routes/appointments.js`): Booking and scheduling
- **/moods** (`routes/moods.js`): Mood tracking
- **/health**: Server status check

## 4. Controllers (Business Logic)
Located in `controllers/`, mapping to the routes above:
- `adminController.js`: Handles admin dashboard data and user administration.
- `authController.js`: Handles registration, login, token generation.
- `doctorController.js`: Handles doctor availability and patient interactions.
- `messageController.js`: Manages sending, receiving, and history of chat messages.
- `patientController.js`: Handles patient profiles, dashboards.
- `postController.js`: Manages community/forum posts.
- `prescriptionController.js`: Logic for creating and retrieving medical prescriptions.

## 5. Database Models (`models/`)
MongoDB schemas mapping entities:
- `User.js`: Base schema for Patients, Doctors, and Admins.
- `Appointment.js`: Scheduled sessions between doctors and patients.
- `Message.js`: Chat messages with sender/receiver references.
- `Mood.js`: Patient mood tracking data.
- `Post.js`: Public posts or articles.
- `Prescription.js`: Medical document linked to patient and doctor.
- `Attempt.js`: Security model for tracking login attempts / rate limiting.

## 6. Key Configurations & Setup
- `server.js`: Entry point, sets up Express middleware, handles Socket.io connections (WebRTC signaling, chat rooms), connects to MongoDB, and defines global error handling.
- Environment variables (`.env`): Contains MongoDB URI, JWT Secrets, Port info, etc.
