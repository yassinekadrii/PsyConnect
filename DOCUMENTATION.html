Technical Project Specifications

The developer's guide to the PsyConnect tele-psychology platform architecture and backend-to-frontend logic flows.

System Architecture

PsyConnect is built with a high-performance stack optimized for medical confidentiality and real-time interaction.

🚀 Node.js / Express
🍃 MongoDB / Mongoose
🔌 Socket.io v4
📹 WebRTC (Peer-to-Peer)

Database Schema

User Model

Identity & Roles

Stores firstName, lastName, email, phone, and password (Bcrypt hashed).

Roles: patient doctor admin

Prescription Model

Medical Records

Links doctor_id to patient_id with nested medicines array.

Keys: name, dosage, duration, notes.

Message Model

Chat Records

Stores 1-to-1 interactions between specialists and users with automatic createdAt indexing for fast retrieval.

Exhaustive API Endpoints

Method Endpoint Description
POST /api/auth/login Authentication via Email/Pass. Returns JWT.
GET /api/admin/dashboard Global platform metrics (Admin only).
POST /api/messages/ Secure real-time message delivery.
POST /api/prescriptions/ Specialist creates a medical record.
DELETE /api/admin/doctors/:id Secure specialist account removal.

Frontend & Real-time Logic

📹 WebRTC & Signaling

The system uses a Signaling Server (Socket.io) to exchange SDP offers. The connection is Peer-to-Peer for maximum privacy.

  • Restricted Call Initiation: Only doctors can start video sessions.
  • Race Condition Guard: Listeners initialize before media prompts.
  • Screen Sharing: Deep integration with replaceTrack API.

🌍 Multi-language Engine

Custom i18n.js handles translation swapping, RTL layout for Arabic, and typography scaling without page reload.

Security Architecture

Bcrypt Hashing: 10 rounds of salt applied via Mongoose pre-save hook.

JWT (JSON Web Token): 8-hour expiry tokens stored securely in LocalStorage.

Role Middleware: Segmented access control (auth, isAdmin, isSuperAdmin).
